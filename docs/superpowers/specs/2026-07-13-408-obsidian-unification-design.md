将 `D:\study\408-考研` 与 `D:\study\408-考研复习` 统一为一套完整的 Obsidian 复习库。最终只保留 `408-考研`，全面重写数据结构、计算机组成原理、操作系统和计算机网络共 26 个章节，并建立可靠的总览、目录、跨科索引、模板和来源说明。

- Obsidian Frontmatter、Wikilink、Callout、Mermaid、LaTeX 和任务清单。

### 不纳入范围

- 不删除或改写 `ocr_output`。
- 不删除或整理 `考研\408` 下的外部参考资料库。
- 不将每个知识点拆成独立原子笔记。
- 不虚构真题年份、题号、频次或尚未发布的 2027 考纲变化。

## 4. 知识来源与优先级

1. 考试政策与范围：教育部、研招网正式信息。
2. 章节主体：`ocr_output` 的 2026 王道 OCR 与 `408-考研` 现有长篇笔记。
3. 复习结构：`408-考研复习` 的 MOC、任务清单、跨科链接和模板。
4. 技术更新：IETF RFC、IEEE/ISO 标准、官方语言或项目文档等一手资料。

正文明确区分：

- **408 必考**：考试口径与作答依据。
- **理解补充**：用于解释原理但通常不直接考查。
- **技术更新**：现实系统与标准的新发展，不覆盖考试标准答案。

OCR 只作为归纳底稿。重写时修正明显错字、断句和识别错误，不大段照录教材。无法由可靠来源确认的信息不写入正文。

## 5. 统一章节结构

每个章节使用统一 Frontmatter：

```yaml
---
title: "章节标题"
subject: "科目"
chapter: 1
exam: 408
status: rewritten
created: YYYY-MM-DD
updated: 2026-07-13
source_version: "王道2026"
tags:
  - 408
  - 科目
aliases:
  - "稳定别名"
---
```

正文结构：

1. 本章定位；
2. 章节导航；
3. 考纲与考点地图；
4. 核心知识框架；
5. 基础概念、核心原理及重点算法、公式或机制；
6. 高频对比表；
7. 典型题型与解题方法；
8. 易错点、记忆方法、典型例题和必要的技术更新；
9. 跨章节与跨科联系；
10. 本章复习清单与自测问题；
11. 资料依据；
12. 前后章节导航。

科目差异：

- 数据结构强化伪代码、复杂度、适用条件和算法设计题。
- 计算机组成原理强化位宽、数据通路、时序、公式与计算步骤。
- 操作系统强化状态转换、调度、PV 操作、内存和文件系统计算。
- 计算机网络强化协议层次、报文结构、时延计算和协议交互。

## 6. 内容质量规则

- 同一结论只在最合适的位置完整解释，其他位置通过 Wikilink 引用。
- 公式使用 LaTeX，并说明变量、单位、前提和边界条件。
- 算法说明输入、输出、核心步骤、复杂度和适用场景。
- 对比内容优先使用表格；状态或交互流程可使用 Mermaid。
- 真题内容只写能够核实的信息，无法核实的题目只归纳为“典型题型”。
- 官方技术资料以链接形式列在章节“资料依据”中，避免长段引用。
- 总览和科目目录同步维护章节状态、重点等级和导航链接。

## 7. 迁移与删除流程

1. 为 `408-考研复习` 的 35 个文件建立迁移映射，逐项标记迁入位置或新版替代文件。
2. 创建统一总览、跨科索引、来源说明和模板。
3. 按数据结构、计算机组成原理、操作系统、计算机网络的顺序重写 26 章。
4. 每完成一科即检查章节覆盖、链接、Frontmatter 和格式，避免问题累积。
5. 全部重写后核对迁移映射，确保旧库的独有内容已有归宿。
6. 验证待删除路径的解析结果严格等于 `D:\study\408-考研复习`。
7. 删除旧库后再次运行完整验收。

删除动作不触碰 `ocr_output`、`408-考研` 或 `考研\408`。当前目录不是 Git 仓库，因此不存在 Git 回退保障；以迁移映射和删除前验证降低风险。

## 8. 验收标准

- 最终业务笔记仅保留 `408-考研` 一套。
- 四科共 26 个章节全部存在并符合统一结构。
- 总览、4 个科目目录、跨科索引、来源说明和 3 个模板完整存在。
- 全部 Markdown 使用 UTF-8。
- 全部业务笔记具有可解析的统一 Frontmatter。
- 内部 Wikilink 的文件目标均可解析；有意保留的未来笔记必须显式标记。
- Markdown 代码围栏、LaTeX 块、Callout 和 Mermaid 结构闭合。
- 标题层级无跳级，章节导航覆盖前后页与科目目录。
- OCR 明显乱码和错误不进入最终正文。
- 每章包含复习清单、自测问题和资料依据。
- 迁移映射中的 35 个旧文件均有明确处理结果。
- 删除后 `D:\study\408-考研复习` 不存在，其他参考目录保持存在。

## 9. 风险控制

- **OCR 错误**：对公式、数字、代码和协议字段做重点复核。
- **教材与现实标准冲突**：考试口径和技术更新分栏表述。
- **网络资料不可靠**：只以一手官方来源支撑关键更新。
- **链接失效**：保留章节文件名并执行自动链接检查。
- **内容丢失**：删除前完成 35 文件迁移映射与验收。
- **任务规模过大**：按科目分批完成和验证，规划文件持续记录进度。
