## 1. 目标

重构 `D:\study` 下的计算机 408、数学一、英语一与大学英语六级资料，使每个目录职责明确、Obsidian 笔记与外部资料分离、OCR 工程可维护，并在可验证的前提下删除重复压缩包和缓存。

本设计采用用户确认的**方案 A：按考试科目分库，资料就近归档**。

本设计必须同时满足：

- 保持 `408-考研` 现有 38 份 Markdown、26 章与 323 个内部链接稳定；
- 将根目录散放的王道 PDF、OCR 脚本和 OCR 输出归入 408；
- 将 `考研/408` 的真题、分章讲义、旧版题库、知识图谱、OneNote 和拓展资料分类迁移；
- 将数学一、英语一纳入统一架构；
- 将误放在英语一中的 CET-6 资料移入独立考试目录；
- 逐项验证后删除已解压 ZIP、缓存和空目录；
- 不删除仅凭名称相似、版本不同或用途不同的资料。

## 6. 迁移与冲突规则

1. 创建目标目录与迁移表，不删除源文件。
2. 记录每个源文件的相对路径、目标路径、字节数和 SHA-256。
3. 从 `CS-Xmind-Note-master.zip` 安全提取唯一缺失的网络层 XMind，拒绝路径穿越。
4. 按映射移动资料；所有目标路径先解析并确认位于 `D:\study`。
5. 目标不存在时移动；目标存在且哈希相同时保留目标并登记重复；目标存在且哈希不同时停止该文件并写入冲突表，绝不覆盖。
6. 更新 Markdown、来源登记与 `.obsidian/workspace.json` 中受影响的路径。
7. 执行迁移后验证；只有验证通过才进入删除阶段。

## 8. OCR 工程安全修正

`batch_ocr.py` 当前含硬编码 API 凭据。迁移时移除明文，改为从 `SILICONFLOW_API_KEY` 环境变量读取；没有变量时以不泄露凭据的错误信息退出。

README 记录：

- 输入 PDF 参数；
- 输出目录约定；
- 环境变量配置；
- 断点页参数；
- 当前本机没有 Python 环境，因此交付只做静态检查，不宣称脚本已运行。

旧凭据可能已经暴露，用户需要在硅基流动控制台撤销并轮换；本地文件重构不能替代服务端轮换。
