# 第4章 网络层

## 本章定位

网络层负责异构网络互连、路由选择与分组转发。本章是地址、分片、路由表和跨层通信流程的最高频综合章。

> IPv4 首部、CIDR/子网、最长前缀匹配、分片、ARP/DHCP/ICMP、RIP/OSPF/BGP、IPv6 基础与路由器。

## 核心知识框架

发送主机先判断目的是否同网段；同网段 ARP 查询目的 MAC，异网段查询默认网关 MAC。路由器解封装二层帧、检查 IP 首部、TTL 减 1、最长前缀匹配、重新封装下一跳帧。端到端 IP 地址通常不变，逐跳 MAC 地址改变。

## 完整知识点

### 网络层服务与路由器

互联网网络层提供无连接、尽最大努力的数据报服务，不保证可靠、顺序或时限。路由选择生成路由信息，分组转发按转发表执行。路由器由输入端口、交换结构、输出端口和路由处理器组成；它隔离广播域。

虚电路网络通信前建立路径，分组携带短 VC 标识，中间结点维护状态；数据报网络每个分组独立携带目的地址并独立选路。

### IPv4 与 CIDR

IPv4 固定首部依次含版本、首部长度、区分服务、总长度、标识、标志、片偏移、TTL、协议、首部检验和、源/目的地址；选项使首部超过 20 B。总长度最大 65535 B。TTL 每经路由器至少减 1；协议字段指明 TCP=6、UDP=17、ICMP=1。

| IPv4 字段 | 位宽 | 单位、作用与 408 边界 |
|---|---:|---|
| 版本 | 4 bit | IPv4 值为 4 |
| IHL | 4 bit | 以 4 B 字为单位；最小 5 即 20 B，最大 15 即 60 B，超过 20 B 的部分为选项与填充 |
| 区分服务 | 8 bit | 经典教材可称服务类型；现代细分不改变首部总位宽 |
| 总长度 | 16 bit | 单位 B，包含首部与数据，最大 65535 B |
| 标识 | 16 bit | 同一原始数据报的各分片取相同标识 |
| 标志 | 3 bit | 保留位、DF（禁止分片）、MF（后有分片）；保留位应为 0 |
| 片偏移 | 13 bit | 单位为 8 B，表示本片数据在原数据报数据区的起始位置 |
| TTL | 8 bit | 每经路由器至少减 1，减为 0 时丢弃 |
| 协议 | 8 bit | 指示上层：ICMP=1、TCP=6、UDP=17 |
| 首部检验和 | 16 bit | 只覆盖 IPv4 首部，TTL 改变后重新计算 |
| 源/目的地址 | 各 32 bit | IPv4 地址；分片过程不改变端点地址 |

> [!info] 理解补充：经典首部与扩展边界
> 408 通常按无选项的 20 B 首部计算。IHL 大于 5 时才加入选项；总长度、分片偏移等字段的位宽不会因选项改变。IPv6 使用扩展首部，不套用 IPv4 的 IHL、首部检验和与路由器分片规则。

地址与掩码按位与得到网络号。前缀 `/n` 有 $32-n$ 个主机位；传统子网可用主机地址数常为 $2^{32-n}-2$。CIDR 用最长前缀匹配；路由聚合要求地址块连续、数量为 2 的幂且起始地址按块大小对齐。

特殊地址：本网络主机位全 0 为网络地址，全 1 为定向广播；`255.255.255.255` 为受限广播；`127.0.0.0/8` 环回；私有地址为 `10/8`、`172.16/12`、`192.168/16`。NAT 转换私网地址，NAPT 还转换端口；它缓解 IPv4 地址不足，但破坏端到端透明性。

若数据报超过 MTU，IPv4 路由器可分片（DF=0）。同一原始数据报各片标识相同；MF=1 表示后面还有片；片偏移以 8 B 为单位，除末片外每片数据长度应为 8 的倍数。每片均有独立 IP 首部，重组只在目的主机进行。

$$
\text{每个非末片最大数据}=\left\lfloor\frac{MTU-H}{8}\right\rfloor\times8
$$

### ARP、DHCP 与 ICMP

- ARP 将同一链路上的 IPv4 地址解析为 MAC。请求广播、回答通常单播；缓存有生存期。跨网发送时解析的是下一跳地址，不是远端主机 MAC。
- DHCP 基于 UDP（服务器 67、客户 68），典型 DORA：Discover（广播）→ Offer → Request（常广播）→ ACK。初始客户源 IP 可为 `0.0.0.0`、目的为受限广播。
- ICMP 封装在 IP 中，用于差错报告与查询。差错报文常含原 IP 首部及其后 8 B，便于定位运输层会话。常见类型：目的不可达、时间超过、参数问题、重定向、回送请求/回答。
- 不对 ICMP 差错报文再发 ICMP 差错；不对分片中的非首片、广播/多播数据报发送常规差错报告。Traceroute 利用 TTL 到期，Ping 利用回送请求/回答。

### 路由算法与协议

| 协议 | 范围/类型 | 交换内容 | 核心算法 |
|---|---|---|---|
| RIP | AS 内，距离向量 | 与邻居周期交换整张距离向量 | Bellman-Ford；跳数最大 15 |
| OSPF | AS 内，链路状态 | 洪泛链路状态，建立相同拓扑库 | Dijkstra；按代价 |
| BGP | AS 间，路径向量 | 可达前缀及 AS-PATH 等属性 | 基于政策选路，TCP 179 |

RIP 收敛慢，存在“好消息快、坏消息慢”，可用水平分割、毒性逆转等缓解。OSPF 使用 Hello 发现邻居，链路状态变化时可靠洪泛，可划分区域。BGP 重视可达性、政策与防环，不等同于求全球最短路径。

转发表查找：候选项中选择匹配位数最多的前缀；无匹配则走默认路由 `0.0.0.0/0`，仍无则丢弃并视条件返回 ICMP。

### IPv6、组播与 SDN

IPv6 地址 128 bit，基本首部固定 40 B，主要字段含版本、流量类、流标签、有效载荷长度、下一个首部、跳数限制、源/目的地址。首部不含检验和；路由器不分片，源主机通过分片扩展首部处理，配合路径 MTU 发现。

> [!info] 技术更新：IPv6
> [RFC 8200](https://www.rfc-editor.org/rfc/rfc8200) 规定 IPv6 基本首部与扩展首部链。408 重点仍是与 IPv4 的字段、地址长度、分片主体和校验差异。

IPv6 文本表示可省去每组前导零，连续全零组可用一次 `::` 压缩；无广播，使用单播、多播和任播。IPv4 到 IPv6 过渡常见双栈与隧道。

IP 组播使用 D 类地址 `224.0.0.0/4`；IGMP 管理主机与本地路由器的组成员关系，组播路由常依赖反向路径转发思想。IPv4 组播 MAC 只映射低 23 bit，因此可能多对一。

> [!info] 理解补充：SDN
> SDN 将控制平面与数据平面分离：控制器集中计算规则，转发设备按流表匹配—动作。北向接口服务应用，南向接口连接转发设备。408 主要掌握概念、平面职责与传统路由对比。

## 典型题型与方法

1. **子网**：地址与掩码按位与求网络号；主机范围排除全 0/全 1；再核对题目是否允许特殊地址。
2. **分片**：先算每片最大数据，按 8 B 对齐，再填总长度、MF、偏移。
3. **最长前缀**：逐项按掩码匹配，选 `/n` 最大者，不按表中先后猜测。
4. **跨网流程**：先判同网，ARP 得下一跳 MAC；每过路由器重建帧并令 TTL 递减。
5. **路由算法**：RIP 做邻居距离加链路代价再取小；OSPF 先建图后跑最短路。

## 易错点

- IPv4 首部检验和只覆盖首部；每跳 TTL 改变后需重算。
- 片偏移单位是 8 B，不是字节；总长度包含片首部。
- ARP 只解决本链路下一跳 MAC，不跨路由器广播。
- DHCP 是应用层协议但直接使用 UDP；ICMP 属网络层并封装在 IP 中。
- BGP 的目标不是单纯最短 AS 路径，政策优先。
- IPv6 路由器不执行分片。
