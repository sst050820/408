72. D
ARP 用于解决局域网上的主机或路由器的 IP 地址和 MAC 地址的映射问题。因此 H4 的 ARP 表中只有和 H4 处于同一虚拟局域网上的主机的 IP 地址和 MAC 地址的映射，H4 位于 VLAN1，H6 位于 VLAN3，因此 D 表示的 H6 的 IP 地址与 MAC 地址的映射不应出现在 H4 的 ARP 表中。

01. 【解答】
数据报长度为 \(4000\text{B}\)，有效载荷为 \(4000 - 20 = 3980\text{B}\)。网络能传送的最大有效载荷为 \(1500 - 20 = 1480\text{B}\)，因此应分为 3 个短些的片，各片的数据字段长度分别为 \(1480, 1480 \div 1020\text{B}\)。片段偏移字段的单位为 8B，\(1480/8 = 185\)，\((1480 \times 2)/8 = 370\)，因此片段偏移字段的值分别为 0、185、370。MF = 1 时，代表后面还有分片；MF = 0 时，代表后面没有分片，因此 MF 字段的值分别为 1、1 和 0（注意，MF = 0 不能确定是独立的数据报，还是分片得来的，只有当 MF = 0 且片段偏移字段 \(>0\) 时，才能确定是分片的最后一个分片）。
02. 【解答】
在 IP 层下面的每种数据链路层都有自己的帧格式，其中包括帧格式中的数据字段的最大长度，这称为最大传输单位（MTU）。\(1500 - 20 = 1480\)，\(2000 - 1480 = 520\)，所以原 IP 数据报经过第一个网络后分成了两个 IP 小报文，第一个报文的数据部分长度是 \(1480\text{B}\)，第二个报文的数据部分长度是 \(520\text{B}\)。
（除最后一个报片外的）所有报片的有效载荷都是 8B 的倍数。\(576 - 20 = 556\)，但 556 不能被 8 整除，所以分片时的数据部分最大只能取 552。第一个报文经过第 2 个网络后，\(1480 - 552 \times 2 = 376 \times 56767 = 5456.27 \times 1020 = 3765672.27\) 个字，则 \((1480 - 552) \times 2 = 376 \times 67567 = 3767672.27\) 个符，因此不使用分片。因此到达目的主机时，原 \(200\text{B}\) 的数据被分成数据长度分别为 \(552\text{B}\)、\(552\text{B}\)、\(576\text{B}\)、\(576\text{B}\)、\(576\text{B}\) 的 4 个小报文。
03. 【解答】
分片的片偏移值表示其数据部分首字节在原始分组的数据部分中的相对位置，单位为 \(8\text{B}\)。首部长度字段以 4B 为单位，总长度字段以字节为单位。题目中，分组的片偏移值为 100，因此其数据部分第一个字节的编号是 800。因为分组的总长度为 100B，首部长度为 \(4 \times 5 = 20\)B，所以数据总分长度为 80B。因此该分组的数据部分最后一个字节的编号是 879。
