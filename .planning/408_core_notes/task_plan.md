## 内容规范

1. 先讲直觉和问题背景，再给定义、算法或协议。
2. 每个核心结论说明适用条件、推导过程和题目识别信号。
3. 算法使用可执行风格伪代码或 C/C++ 示例，并分析时间、空间复杂度。
4. 计算题必须展示单位、位数、边界条件和验算。
5. 检测题答案使用 Obsidian 原生默认折叠 callout，不使用原生 HTML。
6. 原书内容保留 PDF 页码链接；外部资料只作解释和现代扩展。
7. 不使用 `superpower / using-superpowers` 相关 skill 或工作流。
8. 保留并增量完善用户已有 408 笔记，不覆盖 90-复习资料中的第三方知识图谱。

## 错误记录

| 错误 | 次数 | 处理 |
|---|---:|---|
| 并行启动多个 PowerShell 读取 skill 时触发 Windows `CreateProcessWithLogonW 1056` | 1 | 改为单个 PowerShell 进程顺序读取；后续避免并行启动多个本地 shell |
| Adobe PDF 连接器上传 215MB 的计算机网络 PDF 失败 | 1 | 不重复全量上传；改查本机现有 Java/Node/.NET 解析能力，并考虑按页或按对象定向提取 |
| 本机候选 PDF 解析组件搜索命令以 `rg` 无匹配退出码 1 结束 | 1 | 已保留同一命令中成功的环境探测结果；后续对“无匹配”单独容错 |
| 沙箱内 `npm view pdfjs-dist` 网络访问超时 | 1 | 已通过 Mozilla 官方 GitHub releases 核对版本；安装时按网络失败规则请求受控网络权限 |
| 首次 PDF.js inventory 在完成写入后调用了不存在的 `PDFDocumentProxy.destroy()` | 1 | 输出 JSON 已完整生成；删除该清理调用，保留 `cleanup()` 后重新验证退出码 |
| 首次 Windows OCR 脚本直接对 WinRT `IAsyncOperation` 调用 `GetResults()` | 1 | PowerShell 将对象暴露为 COM，改用 `System.WindowsRuntimeSystemExtensions.AsTask<T>` 反射桥接 |
| 章节结构统计在嵌套 `foreach` 后直接接管道，PowerShell 报“空管道元素” | 2 | 改为先收集 `$rows` 再统一输出表格；后续不再对闭合的 `foreach` 直接接管道 |
| 首次同时增补网络第 1、2 章时使用了过长的正文作为补丁上下文，因第 2 章原文细节不完全一致而整体回滚 | 1 | 改为逐章处理，只以稳定的二级标题作为插入锚点 |

| 首次最终验证脚本在双引号字符串中使用 `$k:`，PowerShell 将其误解析为变量名 | 1 | 改为 `${k}` 后重新运行完整验证 |
